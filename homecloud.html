🔧 Step 1

Base System Setup

Install Docker and prepare your system. This is the foundation everything else builds on.

1.1

Install Docker & Docker Compose

Docker containers let us run each service in isolation. This is the cleanest way to manage everything.

💡 These commands work on Ubuntu/Debian. For other distros, check the Docker docs.
bash 
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install Docker using the official script
curl -fsSL https://get.docker.com | sudo sh

# Add your user to the docker group
sudo usermod -aG docker $USER

# Log out and back in, then verify
docker --version
docker compose version

# Enable Docker to start on boot
sudo systemctl enable docker
ℹ️ On macOS, Docker Desktop is the easiest way.
terminal 
# Install Docker Desktop via Homebrew
brew install --cask docker

# Launch Docker Desktop, then verify:
docker --version
docker compose version
⚠️ You need Windows 10/11 Pro for Hyper-V. Home edition users: enable WSL2 instead.
PowerShell (Admin) 
# Enable WSL2
wsl --install

# Restart, then set WSL2 as default
wsl --set-default-version 2

# Install Docker Desktop via winget
winget install -e --id Docker.DockerDesktop

# Verify after restart:
docker --version
☁️ Step 2

Deploy Your Cloud Drive

Set up Nextcloud with a database — your personal Google Drive replacement.

2.1

Create the Docker Compose File

This single file defines all services.

⚠️ Important: Change all passwords below before deploying!
docker-compose.yml 
version: '3.8'

services:
  db:
    image: mariadb:10.11
    container_name: nextcloud-db
    restart: always
    environment:
      - MYSQL_ROOT_PASSWORD=CHANGE_ME!
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=CHANGE_ME!

  nextcloud:
    image: nextcloud:latest
    container_name: nextcloud-app
    restart: always
    ports:
      - "8080:80"
    depends_on:
      - db

  ollama:
    image: ollama/ollama:latest
    container_name: ollama
    restart: always
    ports:
      - "11434:11434"

  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: open-webui
    restart: always
    ports:
      - "3000:8080"
    environment:
      - OLLAMA_BASE_URL=http://ollama:11434
🤖 Step 3

Set Up Your AI Server

Download and run AI models locally. Chat with AI privately, no subscriptions needed.

3.1

Download AI Models with Ollama

Ollama is already running. Now let's download some models.

bash / PowerShell 
# Download a small, fast model
docker exec -it ollama ollama pull phi3:mini

# Download Llama 3.1 8B (recommended)
docker exec -it ollama ollama pull llama3.1:8b

# Download Mistral 7B
docker exec -it ollama ollama pull mistral:7b

# List downloaded models
docker exec -it ollama ollama list
⏱️ Model downloads can be large (2-40 GB). Phi3 Mini is ~2.3 GB and great to start with.
